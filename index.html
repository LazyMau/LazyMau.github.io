Hexo + GitPage 搭建博客

/代码有的是手打，若错误可以复制到google查找正确的。

//参考了

//https://www.jianshu.com/p/f4cc5866946b里面的过程

//仅作记录用

windows下：

在gitbash终端：

1)设置用户名和邮箱

git config --global user.name "your github name"

git config --global user.email "your github sign-in email "

2)生成密钥：

ssh-keygen -t rsa -C "your github sign-in email"

此时如果不用默认的话需要自己创建文件（我是在linux下创建好之后放进去，）

然后添加到GitHub的ssh中

然后测试，如果 ssh -T git@github.com出现Permission denied (publickey)

因为此时ssh私钥文件不是默认的需要自己添加进去

ssh-add /路径/名字（windows下的\需要是双\\）

ubuntu下:

1.安装git

2.配置ssh（配置过的可以不用配置）

1)设置用户名和邮箱

git config --global user.name "your github name"

git config --global user.email "your github sign-in email "

2)生成密钥：

ssh-keygen -t rsa -C "your github sign-in email"

两个回车（不设置密码），成功后

/home/user/用户目录下会多出来.ssh/ 其中包含了私钥文件id_rsa和公钥文件id_rsa.pub

3)

cd ~/.ssh

cat id_rsa.pub

然后将显示出来的文件内容复制到

github->头像->setting->SSH and GPG keys->new SSH key

里面的key文本框，ssh名字自己随意

4）添加成功后可以用

ssh -T git@github.com

验证是否成功

3.创建gitpage仓库

1）.登陆github

2）.以自己用户名.github.io作为仓库名创建仓库

(要安装npm并配置好node.js参考 http://www.runoob.com/nodejs/nodejs-install-setup.html)

4.创建博客文件夹

1)

mkdir blog

cd blog

2)安装hexo

npm install hexo-cli -g

3)初始化Hexo

hexo init

4)安装依赖

npm install

5)启动Hexo

hexo server

启动之后就可以在http：//loaclhost:4000看到示例，ctrl+c可以停止运行。

5.部署Hexo

1.)编辑配置文件，关联远程仓库（github）

`vi _config.yml`



 在deploy:下面添加

 `deploy:`
   ` type: git`
      `repo: git@github.com:yourName/yourName.github.io.git #github仓库地址`

  `branch: master` 



   注意：type、repo、branch的前面有两个空格，后面的:后面有一个空格





     2)退出到博客所在文件夹，安装git插件

     `npm install hexo-deployer-git --save`



      3）`hexo deploy` 或者 `hexo d`



       如果没意外就部署成功了，用http://username.github.io即可访问# Hexo + GitPage 搭建博客
